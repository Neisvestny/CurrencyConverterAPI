# Currency Service API

Backend-сервис для работы с валютами, пользовательскими настройками и кэшированием. Реализует получение курсов валют, хранение данных и оптимизацию через многоуровневый кэш.

| **Автор:** Kishkunou Ruslan

| **Репозиторий:** [CurrencyConverterAPI](https://github.com/Neisvestny/CurrencyConverterAPI)

---

## Возможности

- Получение актуальных курсов валют с внешнего API
- Хранение данных в Supabase
- Кэширование:
    - 24 часа — на уровне базы данных (для пар base-target)
    - 5 минут — в памяти (для одинаковых запросов пользователя)

- Пользовательские настройки:
    - базовая валюта
    - избранные валюты

- Fallback при ошибках API или БД
- Централизованная обработка ошибок через `AppError`
- REST API на Express

---

## Стратегия кэширования

### Кэш в базе данных (24 часа)

- Курсы валют сохраняются в БД
- Каждая пара `(base, target)` кэшируется на 24 часа
- Позволяет уменьшить количество запросов к внешнему API

### Кэш в памяти (5 минут)

- Используется `Map`
- Ключ зависит от пользователя и параметров запроса
- Повторные одинаковые запросы обрабатываются без обращения к БД

### Fallback

Если:

- внешний API недоступен
- база данных возвращает ошибку

→ возвращаются последние доступные данные из кэша

## API

### Получение курсов валют

```
GET /rates?base=USD&targets=EUR,GBP
```

Пример ответа:

```json
{
	"base": "USD",
	"rates": {
		"EUR": 0.92,
		"GBP": 0.78
	}
}
```

---

### Получение пользователя

```
GET /user
```

---

### Обновление настроек пользователя

```
POST /user
```

Тело запроса:

```json
{
	"base_currency": "USD",
	"favorites": ["EUR", "GBP"]
}
```

## Обработка ошибок

Все ошибки возвращаются в едином формате:

```json
{
	"status": "error",
	"code": "USER_NOT_FOUND",
	"message": "User not found",
	"details": {
		"userId": "123"
	}
}
```

### Основные коды ошибок

| Code               | Описание                |
| ------------------ | ----------------------- |
| DATABASE_ERROR     | Ошибка при работе с БД  |
| USER_NOT_FOUND     | Пользователь не найден  |
| VALIDATION_ERROR   | Неверные входные данные |
| EXTERNAL_API_ERROR | Ошибка внешнего API     |

## Заметки по разработке

- Используется `maybeSingle()` для безопасного получения данных
- Все ошибки оборачиваются в `AppError`
- Ключи кэша детерминированы
- Кэш в памяти реализован без Redis
